# C盘清理助手

一个用 PowerShell 写的 Windows C 盘清理脚本。体积非常小，使用或者删除都很方便。

解决了部分在c盘瘦身时不知道删除文件引发后果的问题（可能），该脚本在执行时会附带删除后果的简单说明

# 功能特点

- 默认先扫描，不直接删除
- 高风险操作必须手动确认
- 支持分级清理
- 支持大文件扫描
- 支持保护常见用户数据
- 自动记录清理日志
- 普通模式 / 管理员模式均可运行

---

# 运行方式

建议把脚本放到普通可写目录，例如：

```powershell
D:\工具\C盘清理助手.ps1
```

可以直接鼠标右键脚本使用powershall运行

或者在当前目录打开 PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File ".\C盘清理助手.ps1"
```

如果需要处理：

- Windows 更新缓存
- 系统组件缓存
- 休眠文件
- 系统还原点

建议使用“管理员身份”运行 PowerShell。

---

# 启动保护项

脚本启动时会先询问需要保护哪些内容。

支持的保护项：

| 编号 | 内容 |
|---|---|
| 1 | 保护桌面 |
| 2 | 保护下载目录 |
| 3 | 保护浏览器登录状态和 Cookie |
| 4 | 保护聊天软件数据 |
| 5 | 保护开发环境缓存 |
| 6 | 保护系统还原和回滚能力 |
| 7 | 保护游戏和图形软件缓存 |
| 8 | 大文件只扫描，不自动删除 |

默认配置：

```text
1 3 4 5 6 7 8
```

默认情况下：

- 不会清浏览器登录状态
- 不会碰聊天记录
- 不会删开发环境缓存
- 大文件默认只扫描

下载目录默认不保护。
（因为我经常下载一堆包却忘记删掉）

# 主菜单

扫描完成后会进入主菜单：

```text
[1] 安全清理
[2] 常规清理
[3] 深度清理
[4] 高风险清理
[9] 重新扫描
[0] 退出
```

---

# 选择项目

进入分类后：

选择单个项目：

```text
1
```

一次选择多个项目：

```text
1 3
```

选择当前分类下所有可执行项目：

```text
all
```

返回上一级：

```text
0
```

说明：

- `all` 不会自动选择需要管理员权限的项目
- 仅扫描项目不会被自动执行
- 已跳过项目不会被自动选中

# 清理分类

## 1. 安全清理

主要是常见临时文件：

- 用户临时目录
- Windows 临时目录
- 缩略图缓存
- Windows 错误报告缓存

---

## 2. 常规清理

主要是容易慢慢变大的缓存：

- 回收站
- 浏览器缓存
- Windows 更新下载缓存
- Delivery Optimization 缓存
- 系统日志归档
- 显卡着色器缓存

---

## 3. 深度清理

更偏向“查占用”。

包括：

- Windows 旧组件缓存
- 旧驱动安装包缓存
- 大文件扫描
- 软件缓存目录扫描
- 安装包扫描
- 虚拟机 / 镜像文件扫描

很多项目默认只扫描，不自动删除。

---

## 4. 高风险清理

可能影响：

- 系统回滚
- 登录状态
- 软件缓存

包括：

- 删除休眠文件
- 删除系统还原点
- 删除 Windows.old
- 清理浏览器 Cookie
- 清理聊天软件缓存
- 清理开发环境缓存
- 清理游戏 / 图形软件缓存

高风险项目不会一键执行。

每个项目都必须单独确认，并输入：

```text
YES
```

才会真正执行。

---

# 确认规则

普通项目：

```text
Y
```

或：

```text
y
```

即可执行。

高风险项目：

```text
YES
```

必须大写。

其他输入默认取消。

---

# 权限说明

普通权限下可以处理部分用户级缓存。

下面这些内容通常需要管理员权限：

- Windows 更新缓存
- Delivery Optimization 缓存
- 系统日志归档
- Windows 旧组件缓存
- 删除休眠文件
- 删除系统还原点
- 删除 Windows.old

如果当前不是管理员权限，脚本会自动标记这些项目。

---

# 日志说明

脚本运行过程中不会频繁生成日志。

只有主动退出脚本时，才会统一生成日志文件。

日志目录：

```text
清理日志\YYYY_MM_DD\
```

文件名示例：

```text
C盘清理日志_20260509_153701.txt
```

---

# 失败处理

如果遇到：

- 文件占用
- 权限不足
- 路径不存在

脚本不会直接退出。

失败项目会被跳过，其余项目继续执行。

相关错误会记录到最终日志。

---

# 注意事项

1. 默认目标盘为 C 盘
2. 浏览器缓存目前主要覆盖 Chrome 和 Edge 默认目录
3. 聊天软件路径基于常见默认安装位置
4. 大文件扫描主要用于定位占用来源
5. 不确定影响时，不建议直接执行高风险清理
